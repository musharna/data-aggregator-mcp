# B2 — Croissant 1.1 upgrade (conformsTo + PROV-O provenance + schema.org enrich)

**Status:** PLANNED 2026-06-10. Target v0.28.0. Subagent-driven TDD (impl → spec review → quality review).
**Decision:** upgrade the existing file-level Croissant export (`croissant.py`, currently an un-versioned schema.org
Dataset) to **Croissant 1.1** — add the `conformsTo` version marker, PROV-O provenance populated from our cross-source
enrichment, and the schema.org fields we already hold but don't emit. **ODRL `usageInfo` policy is DEFERRED to B3**
(license-compatibility) where the license→permissions engine lives; B2 emits only a minimal honest `usageInfo` license
pointer, which B3 later upgrades to a full `odrl:Offer`. From the v4 competitive memo
[[data-aggregator-competitive-analysis-v4-2026-06-10]] — contests Eclair on cross-source provenance (the fields a
single-platform Croissant export leaves empty).

## Verified Croissant 1.1 spec (web-fetched 2026-06-10 — docs.mlcommons.org/croissant/docs/croissant-spec-1.1.html)

- **conformsTo URI:** `"http://mlcommons.org/croissant/1.1"` (1.0 was `…/1.0`). Property is `conformsTo` (DCTERMS).
- **@context additions:** `"dct": "http://purl.org/dc/terms/"`, `"prov": "http://www.w3.org/ns/prov#"`,
  `"odrl": "http://www.w3.org/ns/odrl/2/"` (odrl reserved for B3's policy; add the prefix now).
- **PROV-O properties** (apply at dataset level): `prov:wasDerivedFrom` (entity derived from another),
  `prov:wasGeneratedBy` (activity), `prov:wasAttributedTo` (responsible agent).
- **Usage policy:** schema.org `usageInfo` containing an `odrl:Offer` with `odrl:permission`/`odrl:action`/
  `odrl:constraint`. (B2: emit `usageInfo` as a license pointer only — NO fabricated permissions. B3 builds the Offer.)
- **Core props:** `version`, `citeAs` (bibtex), `keywords`, `datePublished`, `dateModified`, `license`, `creator`,
  `publisher`. (No dedicated Croissant `identifier`; schema.org `identifier` applies — we already emit it.)
- **The impl MUST cross-check** the emitted JSON-LD against an official Croissant 1.1 example (the spec page or the
  `mlcommons/croissant` repo `datasets/1.1/` examples) before finalizing — real-execution check for a spec format.

## What our `DataResource` can HONESTLY populate (live-read models.py)

- `creators: list[Creator{name, orcid}]` → **`prov:wasAttributedTo`**: a list of `{"@type":"Person","name":…}` with
  `"@id"` = the ORCID URL when `orcid` present. (Honest: the dataset is attributed to its creators.)
- `links: list[Link{rel, target_id}]` → **`prov:wasDerivedFrom`**: map ONLY clearly-derivation rels →
  `{"@id": <doi-or-url of target_id>}`. Conservative REL set = `{"is_derived_from", "is_version_of",
"is_new_version_of"}` (this entity came from that target). Do NOT map `is_supplement_to`/`cites`/`part_of`
  (those are not derivation — would overstate provenance). If no such rels → omit `prov:wasDerivedFrom`.
- `subjects: list[str]` → **`keywords`** (omit if empty).
- `last_updated` (ISO ts) → **`dateModified`** (omit if None). `year` → `datePublished` (already emitted).
- `source` (e.g. "zenodo") → **`publisher`**: `{"@type":"Organization","name": <pretty source name>}`. Use a small
  source→display-name map (Zenodo, DataCite, GEO, …); fall back to the raw source string.
- `citation` (str, set only when the resolved record was rendered with `cite=`) → **`citeAs`** when present AND it
  looks like bibtex (starts with "@"); else omit. (croissant.render is a PURE transform — it must NOT call the async
  `citation.render`; it only reuses an already-populated `r.citation`.)
- `license` → keep existing `license` property; ALSO emit **`usageInfo`** = the license value (URL or short
  CreativeWork `{"@type":"CreativeWork","name": <license>}`) ONLY when license present. NO `odrl:Offer`, NO asserted
  permissions in B2 — that is B3's job. Add a one-line code comment pointing to B3.

## Build (TDD — `croissant.py` is a 58-line pure transform; extend it)

1. `_CONTEXT`: add `dct`/`prov`/`odrl` prefixes (keep `@vocab` schema.org, `cr`, `sc`).
2. `render(r)`: add `out["conformsTo"] = "http://mlcommons.org/croissant/1.1"` (right after `@type`).
3. Add the PROV-O block: `prov:wasAttributedTo` from creators (helper `_agent(creator)` → Person dict with @id), and
   `prov:wasDerivedFrom` from the conservative derivation-rel links (helper `_derived_from(links)` → list, omit if empty).
4. Add `keywords` (subjects), `dateModified` (last_updated), `publisher` (source map), `citeAs` (bibtex-only guard),
   `usageInfo` (license pointer). All conditional — omit empties, never emit a key with a null/empty value.
5. Update the module docstring: now a Croissant **1.1**-conformant file-level manifest with dataset-level provenance
   (still no RecordSet/Field — that needs file-internal reads; unchanged scope).
6. The `format=croissant` resolve path (`server.py`) is UNCHANGED — it already calls `croissant.render`; the richer
   output flows automatically.

## Spec contracts the reviewers must enforce

- **PROV-O `wasDerivedFrom` is CONSERVATIVE** — only true-derivation rels; never `is_supplement_to`/`cites`/`part_of`.
  Overstating provenance is the cardinal honesty failure for a provenance feature.
- **`usageInfo` is a license POINTER in B2 — no fabricated ODRL permissions** (B3 owns the policy engine).
- **`render` stays a PURE transform** — no I/O, no calling async `citation.render`; reuse `r.citation` only.
- conformsTo must be the EXACT 1.1 URI; @context must carry dct/prov/odrl.
- All new keys CONDITIONAL — never emit a key with empty/null value.
