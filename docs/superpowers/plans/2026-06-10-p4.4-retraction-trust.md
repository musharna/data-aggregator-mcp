# P4.4 — Retraction trust signal Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the first Phase-4 trust signal — a **retraction flag** — surfaced opt-in on `resolve`. This reinforces da-mcp's #1 differentiator (verified-fetch / anti-hallucination): a resolved record can now carry whether its DOI has been **retracted** (per Crossref), so a caller never hands a retracted dataset/paper to a downstream agent unflagged. Scope is **retraction only** (user decision 2026-06-10); CoreTrustSeal `host_certified` and the FAIR proxy sub-score are deferred follow-ups. Releases as **v0.24.0**.

**Architecture (opt-in resolve enrichment, mirrors `cite=`/`format=`):** a new `TrustSignals` pydantic model + a `DataResource.trust: TrustSignals | None = None` field (additive, nullable — the resolve `outputSchema` is `DataResource.model_json_schema()`, so it flows through automatically). A new `trust.py` module exposes `async annotate(client, resource) -> TrustSignals` that makes **one** Crossref `/works/{doi}` call and reads the retraction relation. The `resolve` MCP tool gains an opt-in boolean `trust` arg; when set, the handler calls `annotate` after `router.resolve` and attaches the result via `model_copy` — exactly the pattern `cite`/`format` already use (`server.py:634-649`). No new tool, no new source adapter, no `kind` change.

**Tech Stack:** Python 3.11+, `httpx` (async via `_http.request_json`), `pydantic` v2, `pytest`/`pytest-asyncio` with `httpx.MockTransport` for unit tests and `DATA_AGGREGATOR_MCP_LIVE=1`-gated live probes.

## Grounding — live API shape (probed 2026-06-10 against real DOIs)

**Crossref REST API** (`https://api.crossref.org/works/{doi}`, public, no auth). The retraction relation lives on the **retracted work itself**, under `message.updated-by[]` (NOT `update-to[]` — that is the inverse, carried by the retraction _notice_; an earlier memo had this backwards, which is why the plan re-verified live):

- Retracted (Wakefield, `10.1016/S0140-6736(97)11096-0`): `message.updated-by` = `[{"type":"correction",...}, {"type":"retraction","label":"Retraction","DOI":"10.1016/s0140-6736(10)60175-4","updated":{...}}]`. The title is also prefixed `RETRACTED:` (a secondary, publisher-dependent signal — do NOT rely on it).
- Retracted (Surgisphere/Lancet COVID, `10.1016/S0140-6736(20)31180-6`): `updated-by` types = `['expression_of_concern','correction','retraction','retraction','erratum',...]` → has a `retraction` entry → retracted.
- Clean control (`10.1038/nature14539`): `updated-by` = `[]` → not retracted.
- A DOI Crossref does not register (e.g. a DataCite **data** DOI, or a 404) → **unknown**, not "clean".

**Detection rule (verified):**

- `GET /works/{doi}` → `200` with `message.updated-by[]`: `retracted = any(u.get("type") == "retraction")`; `retraction_doi = that entry's "DOI"`; `concern = any(u.get("type") == "expression_of_concern")`. A found-but-clean work → `retracted=False`, `concern=False` (a real "we checked, it's clean" signal).
- `404` (or the resource has no DOI) → all fields stay `None` (**unknown — never a false negative**). **CORRECTION (Stage-2 review):** this is an opt-in resolve enricher, which by the documented **spec §8** contract (`citation.py:77`, `idconv.py:45`) **never raises into a valid resolve** — so a Crossref outage/timeout/parse-error must ALSO degrade to all-`None` (= unknown, the honest state) with a logged warning, NOT abort the resolve. `resolve(trust=True)` must never be less reliable than resolve without it. (The plan originally said "fail loud" here — wrong; the `cite=`/`format=` enrichers fail-SOFT, and so must this.)

**Crossref etiquette:** the repo sets no `User-Agent` anywhere (httpx default). Send a polite UA from `trust.py` — `data-aggregator-mcp (+https://github.com/musharna/data-aggregator-mcp)` — no personal email. One call per `resolve(trust=True)`; not on search.

**Integration points (verified live in `server.py`/`models.py`):**

1. `server.py:634-649` resolve handler: `resource = await router.resolve(client, args["id"])`, then `cite=args.get("cite")` → `citation.render`, then `fmt=args.get("format")` → croissant/ro-crate, each `resource = resource.model_copy(update={...})`, then `return resource.model_dump()`. The `trust` block slots in right before the `return`.
2. `server.py:23-27` import block aliases enrichment modules (`from data_aggregator_mcp import croissant as croissant_mod`). Mirror: `from data_aggregator_mcp import trust as trust_mod` (alias avoids shadowing the `trust` arg/field).
3. `server.py:379-413` the `resolve` Tool: `inputSchema.properties` has `id`/`cite`/`format`; add `trust`. `outputSchema=DataResource.model_json_schema()` (line 413) is auto-derived → the new field needs no manual schema edit.
4. `models.py:59` `Metrics` is the sub-model precedent (`metrics: Metrics | None = None` on `DataResource`). Add `TrustSignals` right after `Metrics`, and the field on `DataResource` after `metrics`.
5. `_http.request_json(client, method, url, *, service, params=, headers=, timeout=, max_retries=, not_found_returns=)` — pass `not_found_returns=None` so a 404 returns `None` (→ unknown) instead of raising.

```python
"""Trust / integrity signals — retraction status via Crossref.

resolve(trust=True) calls annotate() to attach a TrustSignals to a resolved
resource. The retraction check is ONE Crossref /works/{doi} call: a retracted
work carries its retraction under message.updated-by[] with type=="retraction"
(verified live on real retracted DOIs 2026-06-10 — NOT update-to[], which is the
retraction notice's inverse view). A DOI Crossref doesn't register (e.g. a DataCite
data DOI) 404s → all fields stay None (unknown, NOT a false "clean" claim). Only the
404 path is benign; any other HTTP/parse error fails loud through _http.
"""

from __future__ import annotations

from urllib.parse import quote

import httpx

from data_aggregator_mcp import _http
from data_aggregator_mcp.models import DataResource, TrustSignals

CROSSREF = "https://api.crossref.org/works/{doi}"
_RETRACTION = "retraction"
_CONCERN = "expression_of_concern"
# Crossref polite-pool etiquette: identify the client (no personal email).
_HEADERS = {
    "User-Agent": "data-aggregator-mcp (+https://github.com/musharna/data-aggregator-mcp)",
    "Accept": "application/json",
}
DEFAULT_TIMEOUT = 30.0
MAX_RETRIES = 2


def _doi_of(resource: DataResource) -> str | None:
    return resource.doi or resource.identifiers.get("doi")


async def annotate(client: httpx.AsyncClient, resource: DataResource) -> TrustSignals:
    doi = _doi_of(resource)
    if not doi:
        return TrustSignals()  # nothing to check → unknown
    body = await _http.request_json(
        client,
        "GET",
        CROSSREF.format(doi=quote(doi, safe="/")),
        service="Crossref retraction",
        headers=_HEADERS,
        timeout=DEFAULT_TIMEOUT,
        max_retries=MAX_RETRIES,
        not_found_returns=None,  # 404 → not a Crossref work → unknown
    )
    if not isinstance(body, dict):
        return TrustSignals()
    updated_by = (body.get("message") or {}).get("updated-by") or []
    notice = next(
        (u for u in updated_by if isinstance(u, dict) and u.get("type") == _RETRACTION),
        None,
    )
    concern = any(isinstance(u, dict) and u.get("type") == _CONCERN for u in updated_by)
    return TrustSignals(
        retracted=notice is not None,
        retraction_doi=notice.get("DOI") if notice else None,
        concern=concern,
    )
```
