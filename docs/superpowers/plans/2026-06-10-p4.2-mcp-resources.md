# P4.2 — MCP resources (addressable records) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the MCP **resources** primitive — expose resolved records (and the source catalog) as addressable, readable resources so an MCP client can reference a dataset/record by URI (`@`-mention, link, read) instead of only via the `resolve` tool. This completes the "best-in-class MCP citizen" story (catch-up to servers like cyanheads) and reuses the existing `router.resolve` pipeline. **Resources are a separate MCP primitive — they do NOT count against the tool ceiling.** Releases as **v0.25.0**.

**Architecture (template + catalog over the existing resolve pipeline):** register three handlers on the existing low-level `Server` (`server.py:101`): `@server.list_resources()` (one concrete resource: the source catalog), `@server.list_resource_templates()` (one parameterized template: any record by id), `@server.read_resource(uri)` (parse the URI → return JSON contents). A record URI carries the **same source-prefixed id the `resolve` tool already accepts** (`zenodo:123`, `datacite:10.x`, `pdb:1abc`, a bare Zenodo id, a DOI), URL-encoded; `read_resource` decodes it and calls `router.resolve`. The URI parsing + catalog/template construction live in a new pure `resources.py` module (unit-testable without the MCP transport); `server.py` adds three thin decorated handlers. No new tool, no model change.

**Tech Stack:** Python 3.11+, `mcp` SDK **1.27.2** (low-level `Server`, stdio), `httpx` (async), `pydantic` v2, `pytest`/`pytest-asyncio`.

## Grounding — live MCP SDK 1.27.2 resource contract (read from installed source 2026-06-10)

**Decorators on `mcp.server.lowlevel.server.Server` (all present, verified):**

- `@server.list_resources()` → `async def () -> list[types.Resource]` (the SDK also accepts the new `ListResourcesResult` style; the `list[Resource]` style is supported and simplest).
- `@server.list_resource_templates()` → `async def () -> list[types.ResourceTemplate]`.
- `@server.read_resource()` → `async def (uri: AnyUrl) -> str | bytes | Iterable[ReadResourceContents]`. **Returning `str`/`bytes` is DEPRECATED (emits a warning); return `Iterable[ReadResourceContents]`.** `ReadResourceContents` is `from mcp.server.lowlevel.helper_types import ReadResourceContents` — a dataclass `ReadResourceContents(content: str | bytes, mime_type: str | None = None, meta: dict | None = None)`.

**Types (`mcp.types`, fields verified):**

- `Resource(name, title, uri, description, mimeType, size, icons, annotations, meta)` — `uri` accepts a string (coerced to `AnyUrl`).
- `ResourceTemplate(name, title, uriTemplate, description, mimeType, icons, annotations, meta)` — `uriTemplate` is a plain str (RFC 6570).
- `TextResourceContents(uri, mimeType, text)` / `ReadResourceResult(contents)` — the SDK builds these from the `ReadResourceContents` we return; we never construct them directly.

**Capability advertisement:** `server.run(..., server.create_initialization_options())` (`server.py:729`) auto-derives capabilities from the registered `request_handlers`, so registering the resource handlers **automatically advertises the `resources` capability** — no manual capability wiring.

**URI scheme decision (deviation from the master-plan sketch, with reason):** the master plan wrote `data://record/{source}/{id}`, but **`data:` is the reserved RFC-2397 data-URI scheme** — clients may special-case/try to inline-render it. Use a collision-free custom scheme **`dataresource://`**, and a **single URL-encoded id** (not `source/id` — the id alone, e.g. a DOI `datacite:10.5061/dryad.x`, already contains `:` and `/` which must be encoded anyway). Verified live through `pydantic.AnyUrl`:

- `dataresource://catalog` → `scheme='dataresource'`, `host='catalog'`, `path=None`.
- `dataresource://record/zenodo%3A123` → `host='record'`, `path='/zenodo%3A123'` → `unquote(path.lstrip("/"))` = `zenodo:123`.
- `dataresource://record/datacite%3A10.5061%2Fdryad.x` → `path='/datacite%3A10.5061%2Fdryad.x'` → decodes to `datacite:10.5061/dryad.x`. ✓ (`quote(id, safe="")` then `unquote` round-trips colons + slashes.)

**Integration points (verified live in `server.py`):**

- `server.py:101` `server: Server = Server("data-aggregator-mcp")`; tool/prompt handlers registered as module-level decorators (`@server.list_tools()` 533, `@server.call_tool()` 722).
- Client factory pattern: `async with httpx.AsyncClient(follow_redirects=True) as client:` (`server.py:624,627`).
- `_SOURCES: list[dict[str, Any]]` (`server.py:103`) is the source catalog the `list_sources` tool returns as `{"sources": _SOURCES}` (`server.py:621-623`). The catalog resource returns the same payload.
- `router.resolve(client, rid)` resolves any id shape (`router.py:403`). `DataResource.model_dump_json()` exists (pydantic v2).
- `json` is **NOT** currently imported in `server.py` — add `import json`.

```python
"""MCP resources — addressable, readable records over the resolve pipeline.

Exposes two things to MCP clients as resources (a separate primitive from tools):
  * a single concrete resource, the source catalog, at ``dataresource://catalog``;
  * a template, ``dataresource://record/{id}``, where ``{id}`` is the SAME
    source-prefixed id the resolve tool accepts (zenodo:123, datacite:10.x,
    pdb:1abc, a bare Zenodo id, a DOI), URL-encoded. read_resource decodes it and
    routes through router.resolve.

Custom scheme ``dataresource://`` (NOT ``data://`` — that is the reserved RFC-2397
data-URI scheme clients may special-case). The id is encoded with quote(safe="")
so its colons and slashes round-trip through the URI path.
"""

from __future__ import annotations

from urllib.parse import quote, unquote

from pydantic import AnyUrl

from mcp import types

SCHEME = "dataresource"
CATALOG_URI = "dataresource://catalog"
RECORD_TEMPLATE = "dataresource://record/{id}"


def record_uri(resolve_id: str) -> str:
    return f"{SCHEME}://record/{quote(resolve_id, safe='')}"


def is_catalog(uri: AnyUrl) -> bool:
    return uri.scheme == SCHEME and uri.host == "catalog"


def parse_record_id(uri: AnyUrl) -> str | None:
    """Return the decoded resolve-id for a ``dataresource://record/<id>`` URI,
    else None (not a record URI / empty id)."""
    if uri.scheme != SCHEME or uri.host != "record":
        return None
    raw = (uri.path or "").lstrip("/")
    if not raw:
        return None
    return unquote(raw)


def static_resources() -> list[types.Resource]:
    return [
        types.Resource(
            uri=AnyUrl(CATALOG_URI),
            name="sources",
            title="Data source catalog",
            description="The wired data sources and their capabilities (same payload as the "
            "list_sources tool), as JSON.",
            mimeType="application/json",
        )
    ]


def templates() -> list[types.ResourceTemplate]:
    return [
        types.ResourceTemplate(
            uriTemplate=RECORD_TEMPLATE,
            name="record",
            title="Resolved data record",
            description="Resolve any record by its source-prefixed id (e.g. zenodo:123, "
            "datacite:10.5061/dryad.x, pdb:1abc, a bare Zenodo id, or a DOI) — the same id "
            "the resolve tool accepts, URL-encoded. Returns the full DataResource as JSON.",
            mimeType="application/json",
        )
    ]
```

```python
@server.list_resources()
async def _list_resources() -> list[types.Resource]:
    return resources_mod.static_resources()


@server.list_resource_templates()
async def _list_resource_templates() -> list[types.ResourceTemplate]:
    return resources_mod.templates()


@server.read_resource()
async def _read_resource(uri: AnyUrl) -> list[ReadResourceContents]:
    if resources_mod.is_catalog(uri):
        payload = json.dumps({"sources": _SOURCES})
        return [ReadResourceContents(content=payload, mime_type="application/json")]
    rid = resources_mod.parse_record_id(uri)
    if rid is None:
        raise ValueError(f"not a readable data-aggregator resource: {uri}")
    async with httpx.AsyncClient(follow_redirects=True) as client:
        resource = await router.resolve(client, rid)
    return [ReadResourceContents(content=resource.model_dump_json(), mime_type="application/json")]
```
