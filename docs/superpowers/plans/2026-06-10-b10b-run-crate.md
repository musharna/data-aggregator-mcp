# B10b ‚Äî Whole-search provenance Run Crate (`search(format=provenance)`)

**Status:** PLANNED 2026-06-10. Target v0.34.0. Subagent-driven TDD (impl ‚Üí spec review ‚Üí quality review).
**Decision:** opt-in `search(provenance=true)` that attaches ONE RO-Crate 1.1 "search Run Crate" documenting the whole
search: the RUN itself (query + sources queried + ontology expansions that fired + per-adapter errors) as a schema.org
`CreateAction`, PLUS per-hit provenance for every result record, composing B10a's assessment helpers. The user chose
**per-record dossier FIRST (B10a, shipped v0.33.0) ‚Üí THEN whole-search** ‚Äî this is that follow-up. From the v4 memo
[[data-aggregator-competitive-analysis-v4-2026-06-10]] (B10, Frontier #10). THE "why an aggregator" artifact: one call
yields a machine-readable provenance manifest for an entire search ‚Äî the AI-Act training-data-manifest moment.

## Grounding (reuses shipped code ‚Äî verified live this session)

- B10a's `dossier.py` already renders per-record RO-Crate assessment entities (`_version_result`, `_license_result`,
  `_fair_result`, `_retraction_result`, `_identifier_result`, `_property_value`) and the `CreateAction` pattern, with
  `conformsTo` only on the metadata descriptor (`https://w3id.org/ro/crate/1.1`). B10b REUSES these.
- `router.search_page` returns a `SearchResult{query, total, count, results: list[DataResource], errors: dict,
next_cursor, taxon_/mesh_/tissue_/chemical_/assay_expansion}`. Each result already carries B1 version-currency
  (`is_latest`/`superseded_by`, applied via `_with_version_status`) and `license`. `fair` is NOT attached to search
  hits (it is a resolve enricher) ‚Äî the run crate computes it per hit (pure).
- `server.py` `case "search"` (~L754-772) builds `result` then `return result.model_dump()`. The search inputSchema is
  at `name="search"` (~L308).

## Refactor (dossier.py) ‚Äî make the per-signal helpers reusable with unique @ids

- Add an `id_prefix: str = ""` parameter to each assessment helper so per-hit entities get UNIQUE `@id`s. The `@id`
  becomes `f"#{id_prefix}version-currency"` etc. With the default `id_prefix=""` the `@id`s stay exactly
  `#version-currency`/`#licence`/`#fair`/`#retraction`/`#identifier-chain` ‚Äî **B10a output is byte-identical** (its tests
  must stay green untouched ‚Äî verify).
- Expose a public `assessment_entities(resource, id_prefix="") -> list[dict]` returning the non-None assessment entities
  in order (the same list `dossier.render` already builds). `dossier.render` is refactored to call it (with `id_prefix=""`).
  This is the single reuse seam for `run_crate`.
- **Honesty preserved:** the retraction helper still reads `resource.trust`; for search hits `trust is None` ‚Üí it returns
  None ‚Üí per-hit retraction is OMITTED from the run crate (NOT a negative claim). Document that per-hit retraction is
  available via per-record `resolve(trust=true)`/`resolve(format=provenance)` (B10a) ‚Äî the run crate does not fan out
  N Crossref calls.

## New module: `run_crate.py` ‚Äî PURE renderer over a SearchResult

- `render(result: SearchResult) -> dict[str, Any]` ‚Äî PURE, no network/file I/O, deterministic. (It calls `fair.assess`
  per hit, which is itself pure ‚Äî so `render` stays pure; no Crossref/trust per hit.)
- Structure (one flat RO-Crate 1.1 `@graph`):
  1. metadata descriptor `{"@id": "ro-crate-metadata.json", "conformsTo": {"@id": "https://w3id.org/ro/crate/1.1"},
"about": {"@id": "./"}}` (the ONLY `conformsTo`).
  2. root `{"@id": "./", "@type": "Dataset", "name": f"Search run: {result.query}", "mentions": {"@id": "#search-action"},
"hasPart": [refs to each #hit-i]}`.
  3. the SoftwareApplication agent (reuse `dossier.AGENT_ID`, carrying `__version__`).
  4. a `CreateAction` `#search-action`: `name "data-aggregator-mcp search"`, `instrument` ‚Üí agent, `object` ‚Üí `{"@id": "./"}`,
     and a `query`/result-summary. Encode the RUN provenance HONESTLY:
     - `query` (the user query), `result_count`/`total`,
     - `sources_queried`: the adapters actually queried (derive from the result + `errors` keys ‚Äî be honest about what
       this list means; if the exact adapter set isn't recoverable, document what is encoded),
     - `ontology_expansions`: the echoes that fired (taxon/mesh/tissue/chemical/assay ‚Üí each a small object naming the
       input, the ontology id, and the synonyms added) ‚Äî reuse the `*_expansion` fields,
     - `errors`: the per-source error messages (transparency ‚Äî a partial search is disclosed, not hidden),
     - `result` ‚Üí the list of `#hit-i` refs.
  5. per hit `i` in `result.results`: a `{"@id": f"#hit-{i}", "@type": "Dataset", "name": title, "identifier": doi/url,
"license": ‚Ä¶}` entity, PLUS its assessment entities via `dossier.assessment_entities(hit_with_fair, id_prefix=f"hit-{i}-")`
     where `hit_with_fair = hit.model_copy(update={"fair": fair.assess(hit)})`. Link the hit's assessment set from the
     hit entity (e.g. `"mentions": [refs]`) or from a per-hit CreateAction ‚Äî keep the crate valid and navigable.
- Reuse `dossier`/`ro_crate` constants (AGENT_ID, CONFORMS_TO) rather than re-declaring.

## Models (models.py)

- `SearchResult.provenance_crate: dict[str, Any] | None = None` (after `assay_expansion`). Empty by default; populated
  only when `provenance=true`.

## Server wiring (server.py)

- Add a `provenance` boolean to the `search` inputSchema, documented: opt-in; attaches a `provenance_crate` ‚Äî an RO-Crate
  1.1 Run Crate documenting the query, sources, ontology expansions, errors, and per-hit provenance (version-currency,
  licence+SPDX, FAIR; per-hit retraction is omitted ‚Äî use per-record resolve for that); covers THIS search page only.
- In `case "search"`, after building `result`, before `return`:
  ```python
  if args.get("provenance"):
      result = result.model_copy(update={"provenance_crate": run_crate.render(result)})
  ```
  `import ... run_crate`. (No continuation-cursor concern: the crate documents the page just returned. Document that, on
  a paginated search, each page's crate covers that page ‚Äî stateless, intra-page, mirroring B7.)

## Tests (tests/test_run_crate.py + dossier refactor + search wiring)

- **dossier refactor regression:** `dossier.assessment_entities(r, "")` returns the SAME entities `dossier.render` embeds;
  `dossier.render` output is unchanged (the existing B10a tests pass untouched ‚Äî assert a couple of `@id`s literally).
  `assessment_entities(r, "hit-3-")` yields `@id`s prefixed `#hit-3-‚Ä¶`.
- `run_crate.render` PURE/deterministic (no httpx in its path; twice ‚Üí identical); signature takes only `result`.
- Structural RO-Crate 1.1: `@context`, `@graph`, `conformsTo` ONLY on the descriptor; root Dataset names the query and
  `hasPart`s the hits; the `#search-action` CreateAction has `instrument`/`object`; the agent carries `__version__`.
- Run provenance: the CreateAction encodes the query, the errors dict (a record with a source error ‚Üí it appears), and
  the ontology expansions that fired (a SearchResult with a `mesh_expansion` ‚Üí it shows in the crate); a search with NO
  expansions ‚Üí no fabricated expansion block.
- Per-hit: N results ‚Üí N `#hit-i` Dataset entities, each with version/licence/FAIR assessment entities carrying the hit's
  real values; **per-hit retraction is ABSENT** (trust unattached ‚Üí omitted, asserts nothing negative). An empty result
  set ‚Üí a valid crate with zero hits (no crash).
- Honesty: a hit with `is_latest=None` ‚Üí no version entity for it; a hit with no licence ‚Üí no licence entity; unrecognized
  licence ‚Üí SPDX "unrecognized", never invented (inherited from the dossier helpers ‚Äî assert once end-to-end).
- search wiring: `provenance` in the `search` inputSchema (boolean, not required); `search(provenance=true)` attaches a
  non-None `provenance_crate`; absent/false ‚Üí None; `model_dump()` carries it; the FULL existing search suite passes with
  the flag off (purely additive).
- **Real-execution check (gated `DATA_AGGREGATOR_MCP_LIVE=1`):** run a REAL `search_page(...)` then `run_crate.render`,
  assert a well-formed RO-Crate 1.1 with ‚â•1 hit, per-hit FAIR present, the query encoded, and NO per-hit negative
  retraction claim. Catches real multi-source result-set shapes a fixture hides.

## Explicitly OUT of scope (deferred)

- **Cross-page run crate** ‚Äî one crate spanning all pages of a paginated search; needs state the stateless server lacks.
  Intra-page only; documented. (Same boundary as B7.)
- **Per-hit retraction fan-out** ‚Äî N Crossref calls per search would be slow; per-hit retraction stays a per-record
  opt-in (B10a / `resolve(trust=true)`). Noted in the crate + CHANGELOG.

## Spec contracts reviewers must enforce

- **B10a unchanged** ‚Äî the `id_prefix=""` default keeps `dossier.render` byte-identical; B10a tests pass untouched.
- **Honesty inherited + extended** ‚Äî per-hit retraction OMITTED (never a negative claim from unattached trust); errors
  disclosed (a partial search is shown, not hidden); no fabricated expansion/signal block.
- **Valid RO-Crate 1.1** ‚Äî `conformsTo` only on the descriptor; real `CreateAction`; no invented profile URI.
- **PURE render** ‚Äî deterministic, no network/file I/O (per-hit `fair.assess` is pure; NO per-hit Crossref/trust).
- **Opt-in + additive** ‚Äî default off; with the flag off, search output is byte-identical to pre-B10b.
- **Reuse, don't re-derive** ‚Äî per-hit assessments come from `dossier.assessment_entities`; the base crate idioms from
  `ro_crate`/`dossier` constants.
